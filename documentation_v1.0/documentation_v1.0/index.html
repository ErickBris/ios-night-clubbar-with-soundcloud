How to Customize the App Icon and Other Images#back to top

App Icon

Just open the Assets.xcassets item and inside you have to look for the AppIcon element. Click on it and you will see on the right side the added images. Below each of the pairs, there is the required size in points (pt: 1px = 1pt). If you see below the image a 2x or 3x, this means that you have to multiply the pt value. For better understanding, let's take the first pair (iPhone). There we have 29pt size and two images: one is 2x and the other one is 3x. This means that the 2x has to be 2x29pt = 58pt = 58x58 pixels. The other one (3x) has to be 3x29pt = 87pt = 87x87 pixels. The AppIcon in fact is the pari called iPhone App. There again you have two variants: 2x and 3x and a requirement of 60pt. This means that the 2x should be 120x120 pixels, while the 3x - 180x180 pixels. Now when you know the mathematics, just create your own icons and drag and drop them directly to the relative container - that's it!


Other Image Resources

Within the Assets.xcassets, you can find the rest of the used images. If you want to replace them, just click on the item and then drag and drop your modified image directly to xCode. Please be aware that you have to preserve the orignal's image size. To see the size, right click on the image and click on the Show in Finder - you will get the image's location. Then right click on it and select the Get Info - there you will find the size of the image.

How to Customize the App Name and the Main Content #back to top

App Name

To change the application name, you have to click on the outer NightClubApp item from the left panel and look for the Identity and Type section within the right panel.

Main Content

All of the content is created and maintained via simple XML file, which is located in Assets folder. The name of the file is nightclub.xml

The structure of the file is as simple as possible. We've tried to use semantic elements and this way you could easily understand for what the element is used. Mainly, the content of the app is organized in sections:

  1. <clubinfo> Information about the club, like name, slogan, description, working hours, contact information, social networks, etc. </clubinfo>
  2. <galleries> Information about all galleries </galleries>
  3. <events> Information about all future events that will take place in the club. </events>

-- <clubinfo>

This section contains information about your night club, like name, slogan, description, working hours, phone, address, social networks URLs and soundcloud integration.

It's super easy to adapt this section -- just replace the dummy data with yours and you are done. The only thing which requires more attention is the soundcloud part, but it will be addressed later in the documentation.

-- <galleries>

This section contains all of the gallery items, which you want to display within the app's Gallery screen. As you can see from the XML file, each gallery is created by adding an additional <gallery> </gallery> item, which contains <photos> </photos> item. This <photos> includes as much as you want <photo/> items. Those items, in fact, represent a single image in the gallery.

Example that contains 3 galleris. The first one has 9 images, 2nd one has 2 images, while the 3rd one has one image.

-- -- <gallery>

As you can see from the example, there are several attributes that have to be taken into account. <gallery> has date and cover attributes. Date represents the date from which the images have been shot, while the cover attribute is used to attach a cover image to the gallery. This image will be used within the Gallery screen.

Example:

To add, the cover image, you have to perform two actions: (1) prepare the desired image and place it in GalleryImages folder, which is part of the project - you have to do this outside xCode using Finder (OS X) or Windows Explorer (Windows). (2) Then, open xCode, right click on the GalleryImages folder and click on the Add Files to NightClubApp:

This action will open a dialog from where you have to select the added in step (1) image. This way, you will be able to use this image in the XML file - there you have to point in the cover attribute its name, e.g. cover = "your_new_image.png" The same procedure applies for the rest of the images that you want to use within the XML file.

-- -- -- <photos>

This item has no attributes and acts as a wrapper for the images of the gallery. It can contain several <photo> items.

-- -- -- -- <photo>

This item has two attributes: url and thumb. url is responsible for the big image, which is displayed when a thumbnail image is tapped within the Single Gallery Screen. As you understand, its value can be an image, which in first place has to be added to the GalleryImages folder - you already know how to do it. thumb is responsible for the thumbnail image, which is displayed within the Single Gallery Grid View.

Example:

-- <events>

This section contains all of the events that will take place at your night club.

-- -- <event>

This item represents a single event within the Events screen. It has three attributes (date, starts, ends) and four subelements (<title>, <subtitle>, <description>, <image>). Date attribute is used to set the exact date of the event. Starts and Ends attributes are used to set the start and end time of the event - it couldn't be simpler, right? The subelemnts are used to enter more detailed information about the event like title, subtitle, short description and the image of the event. Please note that the image has to be added to GalleryImages folder and then to the project - you already should know how to do it.

How to Use External Web Hosted XML File #back to top

By default the app is using the local XML file, which is located in Assets folder. There is an option to use external web hosted XML file. This way, there won't be needed to release new version of the app, when make changes to the XML file. You have to be aware that, you will need a web server where to upload the XML file. Then, you have to upload all of the images on the same web server and finally to "tell" the app to use that XML file.

This can be done easily. Just open the : SupportingFiles/AppConstants.swift file and look for the URLPath constant:

As you can see, by default it's set to use the local nightclub XML resource. If you want to change thay. just replace the nightclub value with the URL of your external web hosted XML. For example, our external XML URL is http://dmb-team.com/apps/night-club/nightclub.xml and the constant would look like this: 

                                static let URLPath = "http://dmb-team.com/apps/night-club/nightclub.xml"

Please note that for the images, you have to use their web URL as well. For example, if you want to use image as cover, in first place, you have to upload it to your web server and then to place in the XML file its URL.

How to Integrate SoundCloud with the App #back to top

iOS Night Club App is integrated with SoundCloud. This way, all of your users will be able to listen to music directly within your app. This can be done in few steps...

Sign up - #back to top

The first thing that you have to do is to create an account. To do so, open the SoundCloud Developers website and register an account.

Create a new app - #back to top

When you are ready with the registration, you have to log in and to create new app, clicking the button Register a new application. Enter the desired name and finish the process by clicking on the Register button.

Copy the keys (Client ID and Client Secret) - #back to top

When the app is created, you have to copy the Client ID and Client Secret from SoundCloud. Those keys will be used from the app in order to connect with your SoundCloud app.

Paste the keys in the XML file - #back to top

It's time to paste those keys in the XML file. Just open it and look for the <soundcloudClientID> and <soundcloudClientSecret> items within the <clubinfo> .. </clubinfo> section.

Create a playlist - #back to top

The last step is to create your own playlist, which will be available within the iOS app.To do so, just find or upload tracks and add them to a playlist. It's so easy to add a track to a playlist - just find the track and click on the Add to playlist button. A dialog will be opened, where you will have the option to either Create a playlist or to add the track to already created playlist.

Add as much tracks as you want to the playlist and when you are done, it's time to get the playlist's URL, which has to be entered within the app's XML file.

To do so, click on your profile from within the top navigation and select the Playlist menu item. Then you will see the created playlist (as square). Just click on it's title and the playlist's page will be opened, where you will be able to play all of the tracks that are part of it. Just click on the browser's address bar and copy the playlist's URL.

Then, open the app's XML file and look for the <soundcloud> item within the <clubinfo> section. You have to paste the URL here: 

                            
                            https://soundcloud.com/dmb-team/sets/superhouse-sh-night-club

That's it! Your are done with SoundCloud integration...
